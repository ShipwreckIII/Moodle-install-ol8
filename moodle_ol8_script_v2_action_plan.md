**Current script:** `moodle_install_ol8.sh` (v1 — single monolithic script, verified working with manual fixes)
**Goal:** Rewrite into v2 implementing Fixes #1–#5 below. Fixes #6–#8 are intentionally EXCLUDED.

## Context (what v1 does, verified working on OL 8.10)

Installs Moodle 4.5 on Oracle Linux 8.x:
- PHP 8.2 + MariaDB 10.11 via native DNF module streams (no Remi)
- Apache with DEFAULT DocumentRoot (`/var/www/html`), Moodle served at `/moodle`
  (custom vhost with DocumentRoot=/var/www/html/moodle causes 404 — do not reintroduce)
- PHP sodium extension built from source (no php-sodium package exists on OL8)
- php-fpm must be explicitly enabled (installs disabled; Apache uses proxy_fcgi)
- SELinux stays Enforcing: `httpd_can_network_connect_db` boolean,
  `httpd_sys_rw_content_t` on `/var/moodledata`, `httpd_sys_content_t` on the code
- MariaDB secured non-interactively (scripted SQL, not mysql_secure_installation)
- config.php generated by the script (web installer cannot write it)
- User inputs: SERVER_IP, DB_PASS (moodleuser), ROOT_PASS (MariaDB root)
- NO proxy prompts — servers arrive with proxy pre-configured by network team (NITC);
  curl/dnf use the existing environment automatically

Fixed values (the standard):
- DB: `moodle` / user: `moodleuser` / prefix: `mdl_` / host: localhost
- Moodle dir: `/var/www/html/moodle` | Data dir: `/var/moodledata`
- URL: `http://<SERVER_IP>/moodle`

## Fixes to implement in v2

### Fix #1 — EPEL before libsodium
Add `sudo dnf install -y oracle-epel-release-el8` BEFORE the main package install.
Reason: `libsodium` and `libsodium-devel` live in EPEL; not enabled by default on OL8.
Without this, Step 3 fails: "No match for argument: libsodium libsodium-devel".

### Fix #2 — Use vendored tarballs from the repo (NO downloads)
Both archives are now committed to the GitHub repo:
- `libsodium-2.0.23.tgz` (sodium extension source, ~28 KB)
- `moodle-latest-405.tgz` (Moodle 4.5, ~80 MB)

The script is expected to run FROM the cloned repo directory. Therefore:
- REMOVE the `curl` download of the sodium tarball from pecl.php.net
- REMOVE the `curl` download of Moodle from download.moodle.org
- REPLACE both with local file usage, e.g.:
  `SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"`
  then `cp "${SCRIPT_DIR}/libsodium-2.0.23.tgz" /tmp/` and same for Moodle.
- KEEP the gzip validation (`file ... | grep gzip`) as a sanity check on both files
  (protects against corrupted clones / Git LFS pointer files).
- Reason: proxy env vars are not reliably inherited by script subshells
  (verified failure: `curl: (6) Could not resolve host: pecl.php.net`),
  and vendoring removes all external download dependencies.

### Fix #3 — Resume/state file (skip re-entering values on rerun)
- At startup, before prompting: look for `.install_state.done` then `.install_state.tmp`
  next to the script.
- If found: display the saved SERVER_IP (mask passwords) and ask
  "Reuse saved values? [Y/n]". Y = skip prompts entirely.
- If not found or user declines: prompt as normal, then immediately write
  `.install_state.tmp` containing SERVER_IP, DB_PASS, ROOT_PASS.
- On successful completion (final verification passed): rename
  `.install_state.tmp` -> `.install_state.done`.
- On failure/interruption: file remains `.install_state.tmp` and is picked up next run.
- Set file permissions 600 (contains passwords).

### Fix #4 — Main script + per-step subscripts with resume
Restructure the repo:
```
Moodle-install-ol8/
├── install.sh                  # main orchestrator
├── steps/
│   ├── 01_system_update.sh
│   ├── 02_enable_modules.sh    # php:8.2 + mariadb:10.11 streams
│   ├── 03_install_packages.sh  # includes EPEL first (Fix #1)
│   ├── 04_enable_services.sh   # httpd, mariadb, php-fpm
│   ├── 05_build_sodium.sh      # from vendored tarball (Fix #2)
│   ├── 06_tune_php.sh
│   ├── 07_secure_mariadb.sh
│   ├── 08_create_database.sh
│   ├── 09_extract_moodle.sh    # from vendored tarball (Fix #2)
│   ├── 10_moodledata.sh
│   ├── 11_selinux.sh
│   ├── 12_apache_firewall.sh
│   ├── 13_config_php.sh
│   └── 14_verify.sh            # uses --noproxy '*' (Fix #5)
├── state/                      # .done markers written here (gitignored)
├── libsodium-2.0.23.tgz
└── moodle-latest-405.tgz
```
Main script behavior:
- Each subscript writes `state/NN.done` on success.
- On startup, main script scans `state/` and offers:
  1. Run from start (clears all .done markers — warn first)
  2. Resume from first incomplete step (default when .done markers exist)
  3. Runables from the state file (Fix #3) so each is
  independently runnable.
- `set -e` in every subscript; main script stops on first failing step and
  reports which step failed.

### Fix #5 — Proxy bypass on final verification
Final check must be:
`curl -I --noproxy '*' http://127.0.0.1/moodle/`
Reason: corporate proxy env vars intercept even localhost requests
(verified: plain curl to localhost returned proxy 403 with WAF headers,
while Apache never received the request).
Expected result: HTTP 302/303 with `Location: install.php` (fresh)
or `X-Redirect-By: Moodle` (configured).

## Working rules for the chat doing the rewrite

1. Max 3 steps at a time, wait for confirmation.
2. Exact copy-paste commands with expected output.
3. Troubleshoot before advancing on any error.
4. No assumptions — verify environment first.
5. Detailed comments in every script: what the step does and why
   (including the verified lessons above, so future readers don't
   reintroduce known-broken approaches like Remi, custom vhost,
   PECL install, or curl downloads).
6. User will paste errors from real runs; after each confirmed fix,
   produce a numbered "SCRIPT UPDATE NOTE" for later batching.

## Definition of done

- `git clone` + `./install.sh` on a fresh OL8.x server completes all steps
  with only 3 prompts (IP, DB pass, root pass) — or zero prompts on rerun.
- Final verification returns 302/303.
- Rerun after interruption resumes from the failed step without re-prompting.
- Browser at `http://<SERVER_IP>/moodle` reaches the Moodle web installer.
